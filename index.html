The basics of the Web and HTML

The Web is a collection of documents.
It's sort of like a library. It can keep lots of different things:text, pictures, videos, etc.
But the biggest part of the web is HTML.

HTML is a language that tells the computer how to show all the things on the web.
One way to think about it is to imagine HTML is a hi-tech display case.
It doesn't permanently change the thing you are showing, it just lets you look at it in a different way.
For example, look at these items in their display cases:

Imagine you could shine a light on the cases to emphasize the contents within.
You could slant the cases to the side so we'd have to look at them on an angle.
You could move the middle case to the end so it's in a different position, or move them away from each other so there are big spaces in between each case.
These hi-tech cases even have zoom lenses to make everything inside look bigger or smaller.
But remember, you can always take the item out of the case and it's back to its original state. Then you can put it in a new case to make it look different all over again!

These hi-tech display cases can even show things or bring things that are located in a different case somewhere in the world.
And most amazingly, certain display cases can even bring you to a different room of display cases if you touch it.
We call these special connections references and hyperlinks.

This is what HTML does to all the text, pictures, videos, etc on the web.
All of these things need display cases to show them correctly on the web, so as you can imagine that there is a lot of HTML on the web.

Communicating in HTML

Like all languages, HTML has a vocabulary. And HTML uses this vocabulary to tell the computer how to show things.
And for some things, we tell the computer to start showing something a certain way, but we also need to tell the computer to stop showing it a certain way.
Energizer bunny on a perpetual motion battery.
So we have to be sensitive to how the computer understands HTML. It requires a little different thinking than English or even talking to humans.

The vocabulary of HTML doesn't use words, it uses tags. These are sets of symbols that tell a computer to show something, for example, as
emphasized
bold
or even

spaced

apart.
When we combine these symbols with the thing we want to show, it''s an element
Elements can also be either inline or block.

Imagine writing a school essay or notes on a piece of ruled paper. We usualy write across the page, staying on one line until we run out of room.
Then we go to the next line and keep writing across until we run out of page.
Sometimes are sentences are short so we don't have to go all the way to the end of the page.
But we still start the next sentence on the next line at the beginning and go until we run out of room or don't need anyore room. Then we go to the next line.
HTML Inline Elements are similar becase they like to stick to the lines.
But Block Elements don't stick to the lines. They stick to boxes.

Think about this picture:

If you were taking notes using this paper, would you start writing info in the "cue" section and just write all the way across into the "lecture notes" section?
No, you would get to the line and move your pen back to the left edge of the page before writing any more. The lecture section is for different info.
So block elements are sort of like this, they stay in their boxes, even if we can't visibly see the boxes in our HTML or on the webpage.

HTML has a lot of tags and elements we can use so we can find a lot of different ways to show all the stuff on the web!

Creating a Structured Document

HTML is like the layout of a house. But people just don't want to live in blank houses, even if they are constructed nicely. They also want style.
This is what CSS does. It gives style to a document. CSS may see confusing, but really, it's just another language with a syntax, just like HTML. It just has a different method to a similar goal - Make a great webpage.
Additionally, you can use JavaScript to add interactivity to the house.

So how does this all get put together?
We have all these cool tools - HTML, CSS, JavaScript. How do they work together? Well, like any good team, they need to have some rules and structure to guide them to their goal.

HTML and CSS work together in a Box-like structure. Everything on the page goes in a box, even if it's not blatantly visible when first glancing at a page. Even round things fit into a space of a box.

It may seem like it's a limiting structure, but in fact, it still has many possibilities.
In a webpage, the boxes can go inside other boxes. Like Russian Dolls.
When we put boxes inside boxes, then we arrange the code in a Tree structure, so we don't get confused. It sort of look like a sideways tree, actually. Little triangles represent the branches or elements of the tree.
We can show the division of the boxes using the "div" and "/div" tags. And remember, boxes can go within boxes.

Adding CSS Style

We can use CSS to add style to our HTML in a very efficient way. We can type CSS to apply styles to different areas in just a few instructions. So it can be very easy.

Imagine you live in a house and want to turn certain lights on every night when it gets dark. You could go to each room and turn on the light individually. Or you could put in a special remote control to turn all the lights on in certain rooms in just one command.

CSS is like this. You can make all your pictures have a pink box around them by saying one time, "put a pink box around anything that is labeled a picture", instead of having to go to each picture and write code to tell it to have a pink box around it.

This also reflects an important mantra in computing and programming: Don't Repeat Yourself
The more duplicate instructions you have, the easier it is for you to make a mistake or have something go wrong, and it can be difficult to cleanly delete the instructions. Like Orphan Black.

To enter CSS into an HTML file, you can insert it into the HTML code or write a separate file that you connect to the HTML file. You'll have to decide for yourself which is best for your project since it can depend.
CSS then uses selectors which are used similar to the way tags are used in HTML.
When you enter CSS into your HTML, remember, it will affect everything that you mark to be affected by that particular CSS and if you have more than one CSS selector affecting something at a time, the most specific applies.

The Box Model and Style

The Box model is an important concept to remember. All items in a web page fit into boxes, and boxes can fit inside of boxes.

When we look at giving style to our web page, our boxes can have style too.
The box can be made of four parts:
Margin, Border, Padding, and Content.
Content is whatever the main stuff of a box is - a picture, text, etc.
The Padding goes arond the content, to give it a little space.
The Border goes around the padding and content.
And the Margin keeps your 3-layer box away from other boxes and gives them some space. It has no color.
